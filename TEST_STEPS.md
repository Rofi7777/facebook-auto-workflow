# 📋 测试步骤（详细版）

## 🎯 目标

验证新架构的基础功能是否正常工作，无需后端 API。

---

## 步骤 1: 启动服务器（2分钟）

### 方法 A: 使用提供的脚本（最简单）

在终端运行：

```bash
cd /Users/rofi/Desktop/App/GooGa-Ai-Hub
./start-test-server.sh
```

### 方法 B: 手动启动

```bash
cd /Users/rofi/Desktop/App/GooGa-Ai-Hub
python3 -m http.server 8000
```

**看到这个就成功了**：
```
Serving HTTP on 0.0.0.0 port 8000 ...
```

**保持这个终端窗口打开！**

---

## 步骤 2: 打开浏览器（1分钟）

1. 打开你常用的浏览器（Chrome、Firefox、Safari 都可以）

2. 打开开发者工具：
   - **Mac**: `Cmd + Option + I` 或 `F12`
   - **Windows**: `F12` 或 `Ctrl + Shift + I`

3. 切换到 **Console（控制台）** 标签页

## 步骤 4: 测试集成（5分钟）

在浏览器地址栏输入：
```
http://localhost:8000/test-integration.html
```

1. **点击 "检查模块加载"** 按钮

2. **查看模块状态卡片**：

3. **应该显示**：

1. **点击 "测试组件"** 按钮

3. **应该显示**：
   ```
   ✓ 组件创建成功
   ```

### 4.4 运行完整测试

点击 **"运行完整测试"** 按钮

**应该看到**：
```
✓ 所有集成测试通过 (4/4)
```

## ❌ 如果遇到问题

### 问题 1: 无法访问页面

**检查**:
1. 服务器是否在运行？
2. 端口号是否正确（8000）？
3. 尝试 `http://127.0.0.1:8000` 而不是 `localhost`

### 问题 2: 控制台有错误

**检查**:
1. 错误信息是什么？
2. 是哪个文件出错？
3. 打开 Network 标签页，查看哪些文件加载失败

### 问题 3: 模块未加载

**检查**:
1. 文件是否存在？
   ```bash
   ls -la public/js/core/
   ls -la public/js/services/
   ```
2. 文件路径是否正确？
3. 浏览器 Network 标签页显示什么？
